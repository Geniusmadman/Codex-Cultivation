# Vibe 修仙台规则

## 数据边界

- Codex Desktop 当前没有公开的账号累计 Token 统计接口。修仙台默认只按本主题启用后的输入长度做本地估算，并在 HUD、总览和提示中明确标记。
- 用户已知真实累计值时，可在「修炼总览 → 设置 → 累计 Token 校准」中更新起点。校准用于同步既有修为，不补做历史天劫。
- 状态保存在 renderer 的 `localStorage`：`codex-cultivation-state-v1`。内部 schema 当前为 3，并兼容早期只有 `day/todayTokens/totalTokens` 的 schema 1 和 schema 2。

## 灵石

| 单位 | Token |
| --- | ---: |
| 1 下品灵石 | 10,000 |
| 1 中品灵石 | 1,000,000 |
| 1 上品灵石 | 100,000,000 |
| 1 极品灵石 | 10,000,000,000 |

显示时自动选择不超过当前数值的最高单位；不足 1 下品灵石时仍以下品小数显示。

## 境界

| 境界 | 起点 | 下一大境界 | 小境界 |
| --- | ---: | ---: | --- |
| 炼气 | 0 | 5 亿 | 1-9 层，前期门槛更密集 |
| 筑基 | 5 亿 | 20 亿 | 初期、中期、后期、圆满 |
| 金丹 | 20 亿 | 80 亿 | 初期、中期、后期、圆满 |
| 元婴 | 80 亿 | 320 亿 | 初期、中期、后期、圆满 |
| 化神 | 320 亿 | 1,280 亿 | 初期、中期、后期、圆满 |

筑基以后每个大境界跨度约为前一境界的四倍。四个小境界的分界为当前大境界跨度的 20%、55%、90% 和 100%，让前期推进明显快于后期。

## 顿悟

- 每次有效输入后有 1.2% 概率发现顿悟，触发后至少七天才可能再次出现。
- 顿悟需要用户在总览中主动领取。
- 普通情况下直接到达下一个小境界起点。
- 当前处于大境界最后一个小境界时，只把修为拉到大境界上限并开启天劫，不直接晋级。

## 三日天劫

- 修为到达大境界上限后立即锁定当前境界并开启三日天劫。
- 每日目标取最近七个有修炼记录日的中位数的 80%，向 100 Token 取整，最低为 1,000 Token。
- 从开启当天起连续三天每日达到目标即成功；开启当天已产生的消耗可计入第一日。
- 任一已经结束的目标日未达标即失败。失败扣除当前大境界跨度的 12% 修为，清空天劫期间溢出，需要重新修满后再次渡劫。
- 天劫期间新增 Token 继续计入今日和累计统计，并暂存为溢出修为；渡劫成功后带入下一境界。
- 化神圆满的最终三日天劫成功后标记为「已飞升」。

每个境界最多有两个独立的 2560×1440 无 UI 位图。浅色素材不是深色素材的程序化提亮版。

## 首页布局

- Codex 原生导航保持在最左侧。
- 首页内容区使用三列：修炼面板/仙侍、原生首页核心、修行状态。
- 中央四张卡、项目选择器和输入框始终复用 Codex 原生节点与事件。
- 右侧状态栏先于左侧栏折叠，中央工作区拥有最高空间优先级。
- 右侧只显示本地真实记录或明确标记为本地估算的内容；没有稳定接口的数据不制作伪图表。

## 仙侍

- 性别只在修炼总览的设置面板选择，持久化为 `settings.companionGender`。
- 仙侍卡不显示姓名、等级、修为或独立境界，只显示人物、身份类型和当前角色文案。
- 服装由用户当前大境界自动选择，用户不能手动切衣服。
- 每个性别先建立炼气身份母版；后续筑基、金丹、元婴、化神服装必须以母版做身份保持换装。
- 同一性别固定面孔、发型、体型、年龄感、基础姿态、镜头和构图，只改变服装、配饰与境界特征。
- 境界服装缺失时回退到同性别炼气母版，不回退到另一性别，也不临时程序化绘制人物。

## 本地验证接口

`window.__CODEX_CULTIVATION_DEBUG__` 只用于开发和回归，不在普通 UI 中展示：

- `getState()`：读取状态快照。
- `addTokens(tokens, day?)`：增加指定日期的估算 Token。
- `grantEnlightenment()` / `claimEnlightenment()`：授予并领取顿悟。
- `setState(value)`：写入测试状态并执行 schema 规范化。
- `settle(day?)`：按指定日期结算天劫。
- `resolve()`：读取当前境界、小境界和进度。
