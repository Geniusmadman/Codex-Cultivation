# 修仙工作台视觉系统

## 优先级

`Windows 系统主题 → Codex 官方最终主题 → 修仙工作台视觉层`

修仙工作台只能读取 Codex 已确定的浅色/深色语义色，不得修改系统主题或 `appearanceTheme`。

## 布局

- 官方导航：保持原生结构和交互。
- 左栏：修炼状态与仙侍，桌面宽屏固定展示。
- 中央：今日问道、四个真实功能入口、原生项目选择器和输入框，拥有最高空间优先级。
- 右栏：本地估算状态、日目标、七日节律和最近历程，空间不足时最先折叠。
- 响应式顺序：折叠右栏 → 折叠左栏 → 保留中央工作区。

## 双主题材质

### 浅色

- 白玉、云海、淡金、青瓷、暖白纸面。
- 面板保持足够不透明度，正文对比度至少 4.5:1。
- 阴影偏暖且柔和，边框用淡金与 Codex 官方边框混合。

### 深色

- 玄夜、墨玉、鎏金、冷雾、深青玉光。
- 主文本、次级文本和功能图标分别验证对比度。
- 发光只用于进度、事件和焦点，不制造大面积霓虹。

## 交互

- 所有真实操作继续使用 Codex 原生按钮和输入节点。
- 自定义按钮最小高度 30px；关键触控区域应达到 44px。
- Hover、Focus、Pressed 使用 150–300ms 过渡，支持 `prefers-reduced-motion`。
- 不使用 emoji 充当结构图标；复杂人物与场景使用 AI 位图，基础功能图标复用 Codex/Lucide。

## 数据诚信

- Token、连续天数、天劫、顿悟和历程来自本地状态。
- 日目标必须标记为本地估算，天劫目标除外。
- Codex 未提供稳定接口的工具调用、任务完成数和官方累计 Token 不得伪造。
